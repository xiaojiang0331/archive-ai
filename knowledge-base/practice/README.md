# Australian Bookkeeper & Accountant Practice Knowledge Base

## Scope and status

This is an **initial operating model**, designed for a small Australian bookkeeping/accounting practice that serves small businesses. It describes repeatable work, not universal law or professional advice. Each high-impact action remains subject to the entity, engagement letter, current regulator guidance, and a suitably qualified/registered reviewer.

The library separates:

- **operational facts:** document intake, coding, reconciliation, workpaper preparation;
- **calculation templates:** transparent formulas whose rates/eligibility must be current and approved;
- **regulated decisions:** BAS/tax lodgment, payroll award interpretation, financial statement approval, audit conclusions, AML/CTF assessment, and company secretarial decisions.

## Files

- `task-catalog.json` — machine-readable task cards with inputs, outputs, calculation templates, risk, and reviewer gate.
- `accounting-foundations.md` — accounting concepts, principles, and source hierarchy.
- `operating-manual.md` — human-readable SOP, exception handling, and evidence rules.
- `workflow-maps.md` — standard flow maps, explicitly showing points where practices vary.

## Source hierarchy

The canonical source register is `../sources.json`. This first release uses ATO (BAS/PAYG), Fair Work (records/payslips), TPB (professional conduct and registration), AASB, ASIC, OAIC, AUASB, and AUSTRAC references. Check the linked official source before acting on a date, rate, threshold, eligibility test, or lodgment obligation.

## Publishing rule

Task cards are drafts until a qualified reviewer checks the firm-specific process. The software may use drafts for **internal suggestions and checklists only**, not final client advice or automated lodgment.
